## Quick Setup

<Steps>
  <Step title="Clone the Repository">
    Get the latest version of Walty from GitHub:
    
    ```bash
    git clone https://github.com/ignaciogarcia-dev/walty.git
    cd walty
    ```
  </Step>
  
  <Step title="Configure Environment">
    Copy the example environment file and update the required variables:
    
    ```bash
    cp .env.example .env
    ```
    
    At minimum, you'll need to set:
    
    <ParamField path="DATABASE_URL" type="string" required>
      PostgreSQL connection string. Use `postgresql://wallet:wallet@db:5432/wallet` for Docker setup.
    </ParamField>
    
    <ParamField path="JWT_SECRET" type="string" required>
      Secret for signing JWT tokens. Generate with: `openssl rand -base64 32`
    </ParamField>
    
    <ParamField path="SERVER_PEPPER" type="string" required>
      HMAC key for PIN-based wallet backups. Generate with: `openssl rand -base64 32`
    </ParamField>
    
    <ParamField path="ALCHEMY_API_KEY" type="string" required>
      API key for Alchemy RPC services. Get one at [alchemy.com](https://www.alchemy.com/)
    </ParamField>
    
    <ParamField path="ZEROX_API_KEY" type="string" required>
      API key for 0x swap aggregation. Get one at [0x.org](https://0x.org/docs/api)
    </ParamField>
    
    <Info>
    For a complete list of environment variables, see the [Environment Variables guide](/guides/environment-variables).
    </Info>
  </Step>
  
  <Step title="Start the Stack">
    Build and launch the application with PostgreSQL:
    
    ```bash
    docker compose up --build
    ```
    
    This will:
    - Build the Next.js application image
    - Start PostgreSQL database on port 5432
    - Start the application on port 3000
    - Automatically run database migrations
    
    <Note>
    First build may take 2-3 minutes. Subsequent starts are much faster.
    </Note>
  </Step>
  
  <Step title="Access the Application">
    Open your browser and navigate to:
    
    ```
    http://localhost:3000
    ```
    
    You'll be greeted with the Walty onboarding flow where you can:
    - Create a new wallet with a 24-word seed phrase
    - Import an existing wallet
    - Set up your PIN for quick access
  </Step>
</Steps>

## Verify Installation

<Accordion title="Check running containers">
  ```bash
  docker compose ps
  ```
  
  You should see both `app` and `db` services with status "Up".
</Accordion>

<Accordion title="View application logs">
  ```bash
  docker compose logs -f app
  ```
  
  Watch for any errors during startup. You should see "Ready in X ms" when the app is running.
</Accordion>

<Accordion title="Check database connection">
  ```bash
  docker compose exec db psql -U wallet -d wallet -c "SELECT COUNT(*) FROM users;"
  ```
  
  This verifies the database is accessible and migrations ran successfully.
</Accordion>

## Troubleshooting

<Accordion title="Port 3000 already in use">
  Either stop the conflicting service or change the port mapping in `docker-compose.yml`:
  
  ```yaml
  ports:
    - "3001:3000"  # Use port 3001 instead
  ```
</Accordion>

<Accordion title="Database connection refused">
  Ensure the PostgreSQL container is running:
  
  ```bash
  docker compose ps db
  ```
  
  If not running, check logs:
  
  ```bash
  docker compose logs db
  ```
</Accordion>

<Accordion title="ALCHEMY_API_KEY error">
  Walty requires an Alchemy API key to function. Without it, balance fetching and transaction broadcasting will fail.
  
  1. Sign up at [alchemy.com](https://www.alchemy.com/)
  2. Create an app for Ethereum, Polygon, Arbitrum, Base, and Optimism
  3. Copy your API key to `.env`
  4. Restart the containers: `docker compose restart`
</Accordion>

<Accordion title="Migrations not running">
  Migrations run automatically via the `entrypoint.sh` script. If they fail:
  
  ```bash
  # View migration logs
  docker compose logs app | grep -i migrate
  
  # Manually trigger migration
  docker compose exec app pnpm db:migrate
  ```
</Accordion>

## Development Workflow

If you want to contribute or develop locally without Docker:

```bash
# Start just the database
docker compose -f compose.dev.yml up -d

# Install dependencies
pnpm install

# Run migrations
pnpm db:migrate

# Start dev server
pnpm dev
```

<Info>
For detailed development setup, see the [Development Guide](/developer/development).
</Info>
