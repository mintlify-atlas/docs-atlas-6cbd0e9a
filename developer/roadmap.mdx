## Current Priorities

### 1) Wallet Reliability and Safety

<CardGroup cols={3}>
  <Card title="Recovery Flows" icon="shield-halved">
    Harden unlock/recovery flows
  </Card>
  
  <Card title="Validation & UX" icon="circle-check">
    Improve validation and UX around sensitive actions
  </Card>
  
  <Card title="Pre-flight Checks" icon="plane-departure">
    Strengthen transaction pre-flight checks and error handling
  </Card>
</CardGroup>

### 2) Portfolio Accuracy and Performance

<CardGroup cols={3}>
  <Card title="Balance Consistency" icon="scale-balanced">
    Improve balance and USD valuation consistency across chains
  </Card>
  
  <Card title="Reduce Latency" icon="gauge-high">
    Reduce latency for portfolio refresh and token metadata
  </Card>
  
  <Card title="Provider Fallback" icon="arrows-rotate">
    Improve provider fallback behavior under API/rate-limit errors
  </Card>
</CardGroup>

### 3) Swap Experience

<CardGroup cols={3}>
  <Card title="Quote Reliability" icon="handshake">
    Improve quote reliability and fallback behavior
  </Card>
  
  <Card title="Fee Transparency" icon="dollar-sign">
    Make fee/network-cost feedback clearer before confirmation
  </Card>
  
  <Card title="Edge Cases" icon="triangle-exclamation">
    Improve handling for approval and slippage edge cases
  </Card>
</CardGroup>

### 4) Contributor Experience

<CardGroup cols={3}>
  <Card title="Expand Docs" icon="book">
    Expand docs and onboarding for first-time contributors
  </Card>
  
  <Card title="Good First Issues" icon="flag">
    Add more `good first issue` tasks
  </Card>
  
  <Card title="PR Process" icon="code-pull-request">
    Keep issue templates and PR checks strict and lightweight
  </Card>
</CardGroup>

### 5) POS / Payments

The base invoice/payment-request system is in place (business accounts, QR-code collection, blockchain verification on Polygon). Areas to extend:

<Accordion title="Multi-chain Support">
Allow merchants to accept payments on chains beyond Polygon.
</Accordion>

<Accordion title="Settlement Notifications">
Push or in-app alert when a payment is confirmed.
</Accordion>

<Accordion title="Fiat Amount Locking">
Lock the USDC/USDT amount at request time using live price feeds.
</Accordion>

<Accordion title="Payment History">
Dedicated activity view for received payment requests.
</Accordion>

<Accordion title="Cancellation and Refund Flows">
Allow merchants to cancel or refund payment requests.
</Accordion>

## Proposing New Work

When opening a proposal:

<Steps>
  <Step title="Explain the user problem first">
    Start with the problem you're solving, not the solution.
  </Step>
  
  <Step title="Define scope">
    Clearly state what's in scope and what's explicitly out of scope.
  </Step>
  
  <Step title="Share rollout plan">
    Explain how you'll validate the change and roll it out safely.
  </Step>
  
  <Step title="Mention risks">
    Document any risks and backward-compatibility impact.
  </Step>
</Steps>

For detailed workflow instructions, see the [Contributing Guide](/developer/contributing).
</Info>
