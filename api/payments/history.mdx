<Note>
  This endpoint is only available to **business accounts**. Personal accounts will receive a 403 error.
</Note>

## Overview

Retrieve a paginated list of payment requests created by your business account. Filter by status and use pagination to browse through historical records.

This endpoint returns a simplified view optimized for listing and historical display.

## Authentication

Requires a valid session token in the Authorization header.

```bash
Authorization: Bearer <your-session-token>
```

## Query Parameters

<ParamField query="status" type="string" default="all">
  Filter payment requests by status.
  
  <Expandable title="Available values">
    - `all`: All payment requests (default)
    - `pending`: Only pending requests
    - `confirming`: Only confirming requests
    - `paid`: Only paid requests
    - `expired`: Only expired requests
  </Expandable>
</ParamField>

<ParamField query="limit" type="number" default="50">
  Number of results to return per page.
  
  - Minimum: 1
  - Maximum: 100
  - Default: 50
</ParamField>

<ParamField query="offset" type="number" default="0">
  Number of results to skip (for pagination).
  
  Example: `offset=50` returns results starting from the 51st record.
</ParamField>

## Response

<ResponseField name="items" type="array">
  Array of payment request history items, ordered by creation date (newest first).
</ResponseField>

<ResponseField name="total" type="number">
  Total number of items returned in this response.
  
  <Note>
    This is the count of items in the current page, not the total across all pages.
  </Note>
</ResponseField>

### History Item Object

<ResponseField name="id" type="string">
  Unique identifier for the payment request.
</ResponseField>

<ResponseField name="status" type="string">
  Current status: `pending`, `confirming`, `paid`, or `expired`.
</ResponseField>

<ResponseField name="amountUsd" type="string">
  Payment amount in USD.
</ResponseField>

<ResponseField name="tokenSymbol" type="string">
  Token symbol (`USDC` or `USDT`).
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp when the payment request was created.
</ResponseField>

<ResponseField name="paidAt" type="string | null">
  ISO 8601 timestamp when the payment was confirmed.
  
  `null` if payment has not been completed.
</ResponseField>

<ResponseField name="txHash" type="string | null">
  Transaction hash of the payment.
  
  `null` if no payment detected.
</ResponseField>

<ResponseField name="chainId" type="number">
  Blockchain chain ID. Always `137` (Polygon).
</ResponseField>

## Example

<CodeGroup>

```bash cURL - All Requests
curl "https://walty.app/api/payment-requests/history" \
  -H "Authorization: Bearer YOUR_SESSION_TOKEN"
```

```bash cURL - Paid Requests
curl "https://walty.app/api/payment-requests/history?status=paid&limit=20" \
  -H "Authorization: Bearer YOUR_SESSION_TOKEN"
```

```bash cURL - Pagination
curl "https://walty.app/api/payment-requests/history?limit=25&offset=50" \
  -H "Authorization: Bearer YOUR_SESSION_TOKEN"
```

```javascript JavaScript
const response = await fetch(
  'https://walty.app/api/payment-requests/history?status=paid&limit=20',
  {
    headers: {
      'Authorization': 'Bearer YOUR_SESSION_TOKEN'
    }
  }
);

const { items, total } = await response.json();
```

import requests

response = requests.get(
    'https://walty.app/api/payment-requests/history',
    headers={'Authorization': 'Bearer YOUR_SESSION_TOKEN'},
    params={'status': 'paid', 'limit': 20}

data = response.json()

## Pagination Example

To implement pagination, increment the `offset` by `limit` for each page:

```javascript
async function getAllPaymentRequests() {
  const limit = 50;
  let offset = 0;
  let allItems = [];
  
  while (true) {
    const response = await fetch(
      `https://walty.app/api/payment-requests/history?limit=${limit}&offset=${offset}`,
      {
        headers: { 'Authorization': 'Bearer YOUR_SESSION_TOKEN' }
      }
    );
    
    const { items } = await response.json();
    
    if (items.length === 0) break;
    
    allItems = allItems.concat(items);
    offset += limit;
    
    // Break if we got fewer items than requested (last page)
    if (items.length < limit) break;
  }
  
  return allItems;
}
```

## Filtering by Status

### Active Requests Only

```javascript
// Get pending and confirming requests
const [pendingRes, confirmingRes] = await Promise.all([
  fetch('https://walty.app/api/payment-requests/history?status=pending', {
    headers: { 'Authorization': 'Bearer YOUR_SESSION_TOKEN' }
  }),
  fetch('https://walty.app/api/payment-requests/history?status=confirming', {
    headers: { 'Authorization': 'Bearer YOUR_SESSION_TOKEN' }
  })
]);

const pending = await pendingRes.json();
const confirming = await confirmingRes.json();

const activeRequests = [...pending.items, ...confirming.items];
```

### Completed Payments Only

```javascript
const response = await fetch(
  'https://walty.app/api/payment-requests/history?status=paid&limit=100',
  {
    headers: { 'Authorization': 'Bearer YOUR_SESSION_TOKEN' }
  }
);

const { items: paidRequests } = await response.json();

// Calculate total revenue
const totalRevenue = paidRequests.reduce(
  (sum, req) => sum + parseFloat(req.amountUsd),
  0
);
```

## Analytics Use Cases

### Revenue Tracking

```javascript
async function getRevenueStats() {
  const response = await fetch(
    'https://walty.app/api/payment-requests/history?status=paid&limit=100',
    {
      headers: { 'Authorization': 'Bearer YOUR_SESSION_TOKEN' }
    }
  );
  
  const { items } = await response.json();
  
  const revenue = {
    total: 0,
    byToken: { USDC: 0, USDT: 0 },
    count: items.length
  };
  
  items.forEach(item => {
    const amount = parseFloat(item.amountUsd);
    revenue.total += amount;
    revenue.byToken[item.tokenSymbol] += amount;
  });
  
  return revenue;
}
```

### Conversion Rate

```javascript
async function getConversionRate() {
  const [paidRes, allRes] = await Promise.all([
    fetch('https://walty.app/api/payment-requests/history?status=paid&limit=1000', {
      headers: { 'Authorization': 'Bearer YOUR_SESSION_TOKEN' }
    }),
    fetch('https://walty.app/api/payment-requests/history?limit=1000', {
      headers: { 'Authorization': 'Bearer YOUR_SESSION_TOKEN' }
    })
  ]);
  
  const paid = await paidRes.json();
  const all = await allRes.json();
  
  return {
    conversionRate: (paid.total / all.total) * 100,
    paid: paid.total,
    total: all.total
  };
}
```

## Rate Limits

<Note>
  This endpoint shares the same rate limits as other authenticated API endpoints. Excessive requests may be throttled.
</Note>

Best practices:
- Cache results when possible
- Use appropriate `limit` values to minimize requests
- Avoid polling this endpoint frequently (use [Get Payment Request](/api/payments/get-request) for real-time status)
