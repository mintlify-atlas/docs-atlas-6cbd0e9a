## Overview

Retrieve the list of individual contributions for a split payment request. This endpoint returns all partial payments that have been detected on the blockchain, along with their confirmation status.

This endpoint is public and does not require authentication.

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the payment request.
  
  Example: `clx8k2m9n0000qz7x5y4z1abc`
</ParamField>

## Response

<ResponseField name="contributions" type="array">
  Array of contribution objects, ordered by creation time (oldest first).
</ResponseField>

### Contribution Object

<ResponseField name="id" type="number">
  Unique identifier for this contribution.
</ResponseField>

<ResponseField name="paymentRequestId" type="string">
  ID of the parent payment request.
</ResponseField>

<ResponseField name="txHash" type="string">
  Transaction hash on Polygon blockchain.
  
  Can be viewed on [Polygonscan](https://polygonscan.com) as: `https://polygonscan.com/tx/{txHash}`
</ResponseField>

<ResponseField name="payerAddress" type="string">
  Wallet address of the user who made this contribution.
</ResponseField>

<ResponseField name="amountToken" type="string">
  Amount contributed in token's smallest unit (wei).
  
  For USDC/USDT (6 decimals): divide by 1,000,000 to get human-readable value.
  
  Example: `"30000000"` = 30 USDC
</ResponseField>

<ResponseField name="amountUsd" type="string">
  USD value of the contribution.
  
  Example: `"30.00"`
</ResponseField>

<ResponseField name="tokenSymbol" type="string">
  Token used for this contribution (`USDC` or `USDT`).
</ResponseField>

<ResponseField name="confirmations" type="number">
  Current number of blockchain confirmations.
  
  Increments as new blocks are mined. Maximum tracked value is 2.
</ResponseField>

<ResponseField name="status" type="string">
  Current status of this contribution.
  
  <Expandable title="Status values">
    - `pending`: Transaction detected, waiting for first confirmation
    - `confirming`: Has 1 confirmation, waiting for 2nd
    - `confirmed`: Has reached 2 confirmations
  </Expandable>
</ResponseField>

<ResponseField name="blockNumber" type="string | null">
  Block number where the transaction was included.
  
  `null` if transaction is still pending in mempool.
</ResponseField>

<ResponseField name="detectedAt" type="string | null">
  ISO 8601 timestamp when the contribution was first detected by Walty's monitoring system.
</ResponseField>

<ResponseField name="confirmedAt" type="string | null">
  ISO 8601 timestamp when the contribution reached required confirmations (2).
  
  `null` if still pending or confirming.
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp when the contribution record was created in the database.
</ResponseField>

## Error Responses

| Status Code | Error Message | Description |
|-------------|---------------|-------------|
| 404 | `not found` | Payment request with this ID does not exist |

<Note>
  This endpoint returns contributions for **all** payment requests, whether split or single payment. For non-split payment requests, the array will be empty.
</Note>

## Use Cases

### Tracking Split Payment Progress

```javascript
async function getPaymentProgress(requestId) {
  const [request, contributionsData] = await Promise.all([
    fetch(`https://walty.app/api/payment-requests/${requestId}`).then(r => r.json()),
    fetch(`https://walty.app/api/payment-requests/${requestId}/contributions`).then(r => r.json())
  ]);
  
  const { contributions } = contributionsData;
  const totalPaid = parseFloat(request.totalPaidUsd);
  const totalRequired = parseFloat(request.amountUsd);
  const percentComplete = (totalPaid / totalRequired) * 100;
  
  return {
    percentComplete,
    contributorsCount: contributions.length,
    confirmedContributions: contributions.filter(c => c.status === 'confirmed').length,
    pendingContributions: contributions.filter(c => c.status !== 'confirmed').length
  };
}
```

### Displaying Contributors

```javascript
function displayContributors(contributions) {
  return contributions.map(c => ({
    address: c.payerAddress,
    amount: `$${c.amountUsd} ${c.tokenSymbol}`,
    timestamp: new Date(c.detectedAt).toLocaleString(),
    confirmed: c.status === 'confirmed',
    explorerUrl: `https://polygonscan.com/tx/${c.txHash}`
  }));
}
```

## Blockchain Verification

Each contribution is verified on the Polygon blockchain:

1. **Detection**: System scans for transfers to the merchant's wallet
2. **Validation**: Verifies token type matches the payment request
3. **Confirmation Tracking**: Monitors confirmations from 0 → 1 → 2
4. **Aggregation**: Updates payment request's `totalPaidUsd` when confirmed

<Warning>
  Contributions are only counted toward the payment total after reaching `confirmed` status (2 confirmations). Pending or confirming contributions are tracked but not yet applied.
</Warning>
