## Overview

Retrieve the current status and details of a payment request by its ID. This endpoint is public and does not require authentication, allowing anyone with the ID to check payment status.

For split payments, this endpoint also returns all individual contributions.

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the payment request.
  
  Example: `clx8k2m9n0000qz7x5y4z1abc`
</ParamField>

## Response

Returns the complete payment request object, including real-time status.

<ResponseField name="id" type="string">
  Unique identifier for the payment request.
</ResponseField>

<ResponseField name="status" type="string">
  Current status of the payment request.
  
  <Expandable title="Status values">
    - `pending`: Awaiting payment
    - `confirming`: Payment detected, awaiting blockchain confirmations
    - `paid`: Payment confirmed on blockchain
    - `expired`: Request expired (15 minutes elapsed)
    - `cancelled`: Cancelled by merchant
  </Expandable>
</ResponseField>

<ResponseField name="chainId" type="number">
  Blockchain chain ID. Always `137` (Polygon).
</ResponseField>

<ResponseField name="amountUsd" type="string">
  The requested payment amount in USD.
</ResponseField>

<ResponseField name="tokenSymbol" type="string">
  The token symbol (`USDC` or `USDT`).
</ResponseField>

<ResponseField name="merchantWalletAddress" type="string">
  The merchant's receiving wallet address on Polygon.
</ResponseField>

<ResponseField name="expiresAt" type="string">
  ISO 8601 timestamp when the payment request expires.
</ResponseField>

<ResponseField name="confirmations" type="number">
  Current number of blockchain confirmations.
</ResponseField>

<ResponseField name="requiredConfirmations" type="number">
  Number of confirmations required. Always `2`.
</ResponseField>

<ResponseField name="txHash" type="string | null">
  Transaction hash of the payment. `null` if no payment detected.
  
  For split payments, this is the hash of the final completing transaction.
</ResponseField>

<ResponseField name="isSplitPayment" type="boolean">
  Whether this request accepts split payments.
</ResponseField>

<ResponseField name="totalPaidUsd" type="string">
  Total amount paid so far (split payments only).
</ResponseField>

<ResponseField name="remainingAmountUsd" type="string">
  Remaining amount to be paid (split payments only).
</ResponseField>

<ResponseField name="contributions" type="array">
  Array of individual contributions (split payments only).
  
  <Expandable title="Contribution object">
    <ResponseField name="id" type="number">
      Unique contribution ID.
    </ResponseField>
    
    <ResponseField name="paymentRequestId" type="string">
      ID of the parent payment request.
    </ResponseField>
    
    <ResponseField name="txHash" type="string">
      Transaction hash of this contribution.
    </ResponseField>
    
    <ResponseField name="payerAddress" type="string">
      Wallet address of the contributor.
    </ResponseField>
    
    <ResponseField name="amountToken" type="string">
      Amount contributed in token units (wei).
    </ResponseField>
    
    <ResponseField name="amountUsd" type="string">
      USD value of the contribution.
    </ResponseField>
    
    <ResponseField name="tokenSymbol" type="string">
      Token used for this contribution.
    </ResponseField>
    
    <ResponseField name="confirmations" type="number">
      Blockchain confirmations for this contribution.
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Status: `pending`, `confirming`, or `confirmed`.
    </ResponseField>
    
    <ResponseField name="blockNumber" type="string | null">
      Block number where transaction was included.
    </ResponseField>
    
    <ResponseField name="detectedAt" type="string | null">
      ISO timestamp when contribution was first detected.
    </ResponseField>
    
    <ResponseField name="confirmedAt" type="string | null">
      ISO timestamp when contribution reached required confirmations.
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO timestamp when contribution record was created.
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Responses

| Status Code | Error Message | Description |
|-------------|---------------|-------------|
| 404 | `not found` | Payment request with this ID does not exist |

## Status Transitions

Payment requests follow this status flow:

```
pending → confirming → paid
   ↓
expired (after 15 minutes)
   ↓
cancelled (merchant action)
```

<Note>
  The `status` field automatically reflects expiration. Even if the database shows `pending`, the API returns `expired` if the `expiresAt` timestamp has passed.
</Note>

## Polling Best Practices

When monitoring payment status:

1. **Poll interval**: 3-5 seconds for active requests
2. **Stop conditions**: Stop polling when status is `paid`, `expired`, or `cancelled`
3. **Expiration check**: Monitor time remaining using `expiresAt`
4. **Confirmation tracking**: Watch `confirmations` field increment from 0 → 1 → 2

```javascript
async function pollPaymentStatus(id) {
  const maxAttempts = 300; // 15 minutes at 3s intervals
  
  for (let i = 0; i < maxAttempts; i++) {
    const response = await fetch(`https://walty.app/api/payment-requests/${id}`);
    const payment = await response.json();
    
    if (['paid', 'expired', 'cancelled'].includes(payment.status)) {
      return payment;
    }
    
    await new Promise(resolve => setTimeout(resolve, 3000));
  }
}
```
