## Overview

This endpoint fetches executable swap quotes from both 0x Protocol and 1inch, then returns the best quote based on the highest `buyAmount`. The response includes the complete transaction data needed to execute the swap on-chain.

<Note>
**Quote Aggregation**: Walty queries both 0x and 1inch in parallel and automatically selects the quote that gives you the most tokens for your trade.
</Note>

## Authentication

Requires a valid JWT token in the `Authorization` header:

```
Authorization: Bearer <token>
```

## Query Parameters

<ParamField query="sellToken" type="string" required>
  Token address to sell. Use `"ETH"` or `"MATIC"` for native tokens, or the ERC-20 contract address.
</ParamField>

<ParamField query="buyToken" type="string" required>
  Token address to buy. Use `"ETH"` or `"MATIC"` for native tokens, or the ERC-20 contract address.
</ParamField>

<ParamField query="sellAmount" type="string" required>
  Amount to sell in the token's smallest unit (e.g., wei for ETH)
</ParamField>

<ParamField query="takerAddress" type="string" required>
  The address that will execute the swap transaction
</ParamField>

<ParamField query="chainId" type="number" default="1">
  EVM chain ID (1 = Ethereum Mainnet, 137 = Polygon, etc.)
</ParamField>

## Error Responses

<ResponseField name="error" type="string">
  Error message describing what went wrong
</ResponseField>

### Status Codes

- `200` - Quote retrieved successfully
- `400` - Missing required parameters
- `401` - Unauthorized (invalid or missing token)
- `500` - Quote failed (no liquidity, network error, etc.)

## Implementation Notes

### Quote Aggregation Logic

The endpoint queries both providers simultaneously:

1. **0x Protocol**: Uses the allowance-holder model for gas-efficient swaps
2. **1inch**: Queries v6.0 API with Bearer token authentication
3. **Selection**: Compares `buyAmount` from both quotes and returns the higher one
4. **Fallback**: If one provider fails, returns the successful quote

<Warning>
If both providers fail, the endpoint returns a 500 error with the primary (0x) error message.
</Warning>

### Native Token Handling

- For 0x: Native tokens are represented as `0xeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeee`
- For 1inch: Use the actual wrapped token address or native symbol
- Walty automatically converts `"ETH"` and `"MATIC"` strings to the correct format

### Allowance Requirements

When selling ERC-20 tokens:

1. Check the `issues.allowance` field in the response
2. If `currentAllowance` is less than `sellAmount`, approve the `allowanceTarget` address
3. Use the standard ERC-20 `approve()` function before executing the swap

### Slippage Protection

The quote does not include slippage parameters. For production use:

- Calculate acceptable `buyAmount` with your slippage tolerance (e.g., 0.5%)
- Monitor price changes between quote and execution
- Consider using limit orders for large swaps
