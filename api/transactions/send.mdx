## Overview

This endpoint records a new transaction in the Walty system for tracking purposes. When users send tokens, call this endpoint to create a transaction record with `pending` status. The transaction status can later be updated using the PATCH endpoint.

<Note>
This endpoint only **records** transaction metadata. The actual blockchain transaction must be broadcast separately using your wallet provider.
</Note>

## Authentication

Requires a valid JWT token in the `Authorization` header:

```
Authorization: Bearer <token>
```

## Request Body

<ParamField body="hash" type="string" required>
  The transaction hash from the blockchain
</ParamField>

<ParamField body="fromAddress" type="string" required>
  The sender's wallet address
</ParamField>

<ParamField body="toAddress" type="string" required>
  The recipient's wallet address
</ParamField>

<ParamField body="tokenSymbol" type="string" required>
  Token symbol (e.g., "ETH", "USDC", "MATIC")
</ParamField>

<ParamField body="value" type="string" required>
  Transaction amount in token's smallest unit (wei for ETH)
</ParamField>

<ParamField body="chainId" type="number" default="1">
  EVM chain ID (1 = Ethereum, 137 = Polygon, etc.)
</ParamField>

<ParamField body="chainType" type="string" default="EVM">
  Blockchain type, currently only "EVM" is supported
</ParamField>

<ParamField body="tokenAddress" type="string">
  Token contract address (null for native tokens like ETH)
</ParamField>

## Response

<ResponseField name="ok" type="boolean">
  Returns `true` if the transaction was successfully recorded
</ResponseField>

## Error Responses

<ResponseField name="error" type="string">
  Error message describing what went wrong
</ResponseField>

### Status Codes

- `200` - Transaction recorded successfully
- `400` - Missing required fields
- `401` - Unauthorized (invalid or missing token)

## Implementation Notes

- All transactions are initially created with `status: "pending"`
- The `value` field should be a string representing the amount in the token's smallest unit
- For native tokens (ETH, MATIC), set `tokenAddress` to `null` or omit it
- The transaction hash must be unique across all users
- Use the [Update Transaction Status](/api/transactions/history) endpoint to mark transactions as confirmed or failed
