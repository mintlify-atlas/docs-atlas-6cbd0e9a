---
title: "Get Addresses"
api: "GET /api/addresses"
---

Retrieves all wallet addresses associated with the authenticated user's account.

## Authentication

This endpoint requires authentication via JWT cookie.

## Response

<ResponseField name="addresses" type="array">
  Array of address records belonging to the authenticated user.
  
  <Expandable title="Address">
    <ResponseField name="id" type="number">
      Unique identifier for this address record
    </ResponseField>
    
    <ResponseField name="userId" type="number">
      The user ID this address belongs to
    </ResponseField>
    
    <ResponseField name="address" type="string">
      The wallet address (EVM format)
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Responses

<ResponseField name="error" type="string">
  Error message when request fails
</ResponseField>

**Status Codes:**
- `401` - Unauthorized (invalid or missing JWT cookie)

## Examples

<CodeGroup>

```bash cURL
curl -X GET "https://api.walty.io/api/addresses" \
  -H "Cookie: auth-token=YOUR_JWT_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.walty.io/api/addresses', {
  credentials: 'include', // Include cookies
});

const data = await response.json();
console.log(data.addresses);
```

</CodeGroup>

### Success Response

```json
{
  "addresses": [
    {
      "id": 1,
      "userId": 42,
      "address": "0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb"
    },
    {
      "id": 2,
      "userId": 42,
      "address": "0x1234567890123456789012345678901234567890"
    }
  ]
}
```

### Error Response

```json
{
  "error": "Unauthorized"
}
```

<Note>
  The database schema enforces a unique constraint on `(userId, address)` pairs, preventing duplicate addresses for the same user.
</Note>
