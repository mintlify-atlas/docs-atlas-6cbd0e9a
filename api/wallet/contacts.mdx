---
title: "Manage Contacts"
---

Manage your wallet contacts - addresses you frequently interact with. These endpoints allow you to list, add, and remove contacts.

## List Contacts

<api>GET /api/contacts</api>

Retrieves all contacts for the authenticated user.

### Authentication

This endpoint requires authentication via JWT cookie.

### Response

Returns an array of contact objects.

<ResponseField name="id" type="number">
  Unique identifier for this contact
</ResponseField>

<ResponseField name="userId" type="number">
  The user ID this contact belongs to
</ResponseField>

<ResponseField name="name" type="string">
  Display name for the contact
</ResponseField>

<ResponseField name="address" type="string">
  The wallet address associated with this contact
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO timestamp when the contact was created
</ResponseField>

### Error Responses

**Status Codes:**
- `401` - Unauthorized (invalid or missing JWT cookie)

### Examples

<CodeGroup>

```bash cURL
curl -X GET "https://api.walty.io/api/contacts" \
  -H "Cookie: auth-token=YOUR_JWT_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.walty.io/api/contacts', {
  credentials: 'include',
});

const contacts = await response.json();
```

</CodeGroup>

#### Success Response

```json
[
  {
    "id": 1,
    "userId": 42,
    "name": "Alice",
    "address": "0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb",
    "createdAt": "2026-03-10T15:30:00.000Z"
  },
  {
    "id": 2,
    "userId": 42,
    "name": "Bob's Wallet",
    "address": "0x1234567890123456789012345678901234567890",
    "createdAt": "2026-03-11T09:15:00.000Z"
  }
]
```

## Add Contact

<api>POST /api/contacts</api>

Creates a new contact for the authenticated user.

### Authentication

This endpoint requires authentication via JWT cookie.

### Request Body

<ParamField body="name" type="string" required>
  Display name for the contact
</ParamField>

<ParamField body="address" type="string" required>
  The wallet address to save
</ParamField>

### Response

Returns the created contact object.

<ResponseField name="id" type="number">
  Unique identifier for the newly created contact
</ResponseField>

<ResponseField name="userId" type="number">
  The user ID this contact belongs to
</ResponseField>

<ResponseField name="name" type="string">
  Display name for the contact
</ResponseField>

<ResponseField name="address" type="string">
  The wallet address
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO timestamp when the contact was created
</ResponseField>

### Error Responses

<ResponseField name="error" type="string">
  Error message when request fails
</ResponseField>

**Status Codes:**
- `400` - Missing required fields (name or address)
- `401` - Unauthorized (invalid or missing JWT cookie)

### Examples

<CodeGroup>

```bash cURL
curl -X POST "https://api.walty.io/api/contacts" \
  -H "Cookie: auth-token=YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Alice",
    "address": "0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.walty.io/api/contacts', {
  method: 'POST',
  credentials: 'include',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'Alice',
    address: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb',
  }),
});

const contact = await response.json();
```

</CodeGroup>

#### Success Response

```json
{
  "id": 3,
  "userId": 42,
  "name": "Alice",
  "address": "0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb",
  "createdAt": "2026-03-13T12:00:00.000Z"
}
```

#### Error Response

```json
{
  "error": "Missing fields"
}
```

## Delete Contact

<api>DELETE /api/contacts</api>

Removes a contact from the authenticated user's list.

### Authentication

This endpoint requires authentication via JWT cookie.

### Request Body

<ParamField body="id" type="number" required>
  The ID of the contact to delete
</ParamField>

### Response

<ResponseField name="ok" type="boolean">
  Returns `true` when deletion is successful
</ResponseField>

### Error Responses

<ResponseField name="error" type="string">
  Error message when request fails
</ResponseField>

**Status Codes:**
- `400` - Missing id field
- `401` - Unauthorized (invalid or missing JWT cookie)

### Examples

<CodeGroup>

```bash cURL
curl -X DELETE "https://api.walty.io/api/contacts" \
  -H "Cookie: auth-token=YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"id": 3}'
```

```javascript JavaScript
const response = await fetch('https://api.walty.io/api/contacts', {
  method: 'DELETE',
  credentials: 'include',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({ id: 3 }),
});

const result = await response.json();
```

</CodeGroup>

#### Success Response

```json
{
  "ok": true
}
```

#### Error Response

```json
{
  "error": "Missing id"
}
```

<Note>
  The deletion is user-scoped - you can only delete your own contacts. The query uses `AND` conditions on both `id` and `userId` to ensure security.
</Note>
